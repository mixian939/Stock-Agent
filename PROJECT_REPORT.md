## 一、项目概述

本项目是一个**基于 LLM Agent 的 ETF 动量轮动量化交易系统**。核心思路是利用动量因子在 7 只 ETF 之间轮动配置，由 LLM Agent 自主分析行情并执行买卖决策。系统包含完整的回测引擎、策略模块、Agent 决策层、Web 仪表盘和结构化日志，是一个从数据获取到可视化展示的全链路量化系统。

## 三、策略详解：动量轮动

### 3.1 什么是动量因子

动量（Momentum）是量化投资中最经典的因子之一。核心假设是：**过去一段时间表现强势的资产，未来一段时间大概率继续强势**（趋势延续效应）。

学术上，Jegadeesh & Titman (1993) 的经典论文证实了 3-12 个月动量效应的显著性。

### 3.2 动量计算公式

```
动量 = 今日收盘价 / 20个交易日前收盘价 - 1
```

使用 20 个交易日（约一个自然月）作为回看窗口。例如：
- 某 ETF 20 天前收盘价 4.000，今日收盘价 4.200
- 动量 = 4.200 / 4.000 - 1 = **+5.00%**

### 3.3 轮动规则（逐步说明）

每个**周五**（调仓日），系统执行以下逻辑：

```
第一步：计算所有 7 只 ETF 的 20 日动量值
第二步：按动量从高到低排名
第三步：取排名前 2 名（TOP_N = 2）
第四步：判断配置方案
   ├── 若前 2 名的动量全部 ≤ 0  →  100% 国债ETF（全面避险）
   ├── 若前 2 名动量全部 > 0    →  各 50% 等权配置
   └── 若只有 1 只动量 > 0      →  该 ETF 50% + 国债ETF 50%
第五步：单只 ETF 最高不超过 50%，超出部分分给国债ETF
第六步：先卖后买，执行调仓
```

### 3.4 风控机制

**回撤止损**：每个交易日实时监控组合净值回撤。当净值从峰值回撤超过 **-8%** 时：
1. 立即清仓所有持仓（紧急平仓）
2. 进入 **10 个交易日** 的冷却期（全部持有现金）
3. 冷却期结束后重置峰值，恢复正常交易

**交易成本控制**：
- 佣金费率：万分之三（0.03%）
- 最小交易单位：100 股（A股规则）
- 买入时自动调整为可负担的最大整手数

### 3.5 策略参数汇总

| 参数 | 值 | 含义 |
|------|-----|------|
| MOMENTUM_WINDOW | 20 交易日 | 动量回看窗口 |
| TOP_N | 2 | 持有动量前 N 名 |
| REBALANCE_WEEKDAY | 周五 | 调仓日 |
| MAX_SINGLE_WEIGHT | 50% | 单只最大仓位 |
| MAX_DRAWDOWN_STOP | -8% | 回撤止损线 |
| STOP_COOLDOWN_DAYS | 10 交易日 | 止损冷却期 |
| COMMISSION_RATE | 0.03% | 交易佣金 |
| INITIAL_CAPITAL | 100 万元 | 初始资金 |

## 四、LLM Agent 决策机制

### 4.1 Agent 架构

系统使用 **Agno** 框架构建 Agent，连接 OpenAI 兼容的 LLM 接口。Agent 拥有 8 个可调用的工具（Tool Calling / Function Calling）：

| 工具名 | 功能 |
|--------|------|
| `get_current_date` | 获取当前模拟日期和星期 |
| `get_market_data` | 查看指定 ETF 近期行情（OHLC + 涨跌幅） |
| `get_momentum_rankings` | 计算并查看所有 ETF 的动量排名 |
| `get_portfolio_status` | 查看当前持仓、现金、总资产、回撤 |
| `get_recommended_allocation` | 获取策略推荐的目标配置 |
| `execute_rebalance` | 执行调仓（附带决策理由） |
| `get_performance_summary` | 查看回测绩效指标 |
| `check_if_rebalance_day` | 检查今天是否为调仓日 |

### 4.2 Agent 决策流程

在 Agent 回测模式下，每个周五系统会发给 Agent 如下指令：

> "今天是 YYYY-MM-DD（调仓日）。请分析当前市场动量并执行调仓。
> 1. 查看动量排名
> 2. 获取策略推荐
> 3. 查看当前持仓
> 4. 执行调仓
> 给出你的分析和决策理由。"

Agent 自主调用工具、分析数据、给出中文分析报告，最终执行交易。这与传统量化系统的纯算法执行不同——**Agent 可以理解市场语境、解释决策逻辑、回应异常情况**。

### 4.3 Headless vs Agent 模式对比

| 维度 | Headless 模式 | Agent 模式 |
|------|--------------|-----------|
| 决策方 | 纯算法自动执行 | LLM 自主分析后执行 |
| 确定性 | 完全确定（同数据同结果） | 非确定性（LLM 输出有随机性） |
| 速度 | 秒级完成 | 每次调仓需等待 LLM 响应 |
| 可解释性 | 无（仅记录权重） | 强（Agent 输出分析文本） |
| 适用场景 | 快速回测验证 | 展示 AI 决策能力 |

## 六、技术实现亮点

### 6.1 未来数据泄漏防护

回测系统最容易犯的错误是使用了未来数据（look-ahead bias）。本系统在 `MarketFeed` 中实现了严格的防泄漏机制：

```python
# 只返回当前模拟日期及之前的数据
df[pd.to_datetime(df["trade_date"]) <= self.current_date]
```

通过游标（cursor）逐日推进，策略在任何时刻只能看到"当天及之前"的行情数据。

### 6.2 策略与 Agent 共用同一套逻辑

Agent 的 8 个工具底层调用的是与 Headless 模式完全相同的策略函数（`rank_etfs`、`generate_target_weights`）。这保证了两种模式的策略一致性——Agent 看到的推荐和 Headless 执行的结果完全一致。

### 6.3 先卖后买的调仓顺序

`Portfolio.rebalance_to()` 严格遵循**先卖后买**的顺序。这确保了卖出释放的资金可以用于买入，最大化资金利用率，避免因现金不足而无法完成目标配置。

### 6.4 结构化日志

每次回测生成 3 个 JSON 文件：
- `trades_*.json`：每笔交易的完整记录（方向、股数、价格、佣金）
- `decisions_*.json`：每次调仓的动量排名、目标权重、决策理由
- `daily_*.json`：每个交易日的净值、现金、持仓、回撤

支持回溯分析和问题排查。

### 6.5 全栈可视化

通过 FastAPI + Next.js 实现了实时仪表盘，包含：
- 净值曲线图（Recharts LineChart）
- 回撤面积图（Recharts AreaChart）
- 持仓饼图（Pie Chart）
- 绩效指标卡片（6项核心指标）
- 交易记录表格（最近30笔）

## 七、策略优缺点分析

### 7.1 优点

| 优点 | 说明 |
|------|------|
| **逻辑简洁清晰** | 动量因子有坚实的学术基础（Jegadeesh & Titman 1993），策略规则简单可解释 |
| **跨资产轮动** | 7只ETF覆盖股票、债券、商品、海外，天然分散风险 |
| **自动避险机制** | 当所有资产动量为负时自动切入国债，避免在熊市中硬扛 |
| **回撤控制** | -8%止损线 + 10天冷却期，限制了最大亏损 |
| **执行纪律** | 算法化执行消除了人为情绪干扰（贪婪/恐惧） |
| **交易频率适中** | 周频调仓，既能跟上趋势又不至于过度交易 |
| **Agent 可解释** | LLM Agent 能用自然语言解释每次决策的逻辑 |

### 7.2 缺点与局限

| 缺点 | 说明 |
|------|------|
| **动量反转风险** | 动量策略的天然敌人是"反转"——强势资产突然暴跌。20日窗口可能在趋势反转初期仍然给出正信号 |
| **回测区间较短** | 仅 6 个月数据，无法验证策略在不同市场环境（牛市/熊市/震荡市）中的表现 |
| **过拟合风险** | 参数（窗口20天、TOP_N=2、周五调仓等）在当前数据上表现好，不代表在其他时段同样有效 |
| **未考虑滑点** | 真实交易中存在买卖价差（滑点），尤其是大额交易对小盘 ETF 的冲击成本 |
| **ETF 池固定** | 7只ETF是手动选定的，没有动态筛选机制。如果某只ETF退市或流动性恶化，系统无法自适应 |
| **单因子策略** | 仅使用动量一个因子，未结合估值、波动率、基本面等多因子模型 |
| **止损后的机会成本** | 冷却期内全部持有现金，可能错过急跌后的反弹行情 |
| **Agent 模式的成本** | LLM 调用有延迟和费用，且非确定性输出可能导致不同次回测结果不一致 |

### 7.3 改进方向

| 方向 | 具体措施 |
|------|----------|
| 多因子增强 | 在动量基础上加入波动率调整（风险平价）、估值因子 |
| 参数鲁棒性 | 进行参数扫描（窗口5~60天、TOP_N 1~4），验证策略稳定性 |
| 更长回测 | 扩展至 3~5 年数据，覆盖 2020 疫情、2021 牛市、2022 熊市等不同环境 |
| 滑点模型 | 根据 ETF 日均成交量估算冲击成本 |
| 动态止损 | 使用 ATR（平均真实波动幅度）动态调整止损线，而非固定 -8% |
| 基准对比 | 加入沪深300基准线，直观展示超额收益 |

## 八、项目技术架构详解

### 8.1 模块依赖关系

```
config.py（全局配置）
    ↓
data/fetcher.py → data/feed.py（数据获取 → 市场模拟）
    ↓
strategy/momentum.py（策略计算，纯函数无状态）
    ↓
engine/portfolio.py + engine/performance.py（组合管理 + 绩效追踪）
    ↓
backtest/simulator.py（回测主循环，协调以上所有模块）
    ↓
agent/tools.py + agent/trading_agent.py（Agent 工具桥接 + LLM Agent）
    ↓
server/app.py + server/dashboard_routes.py（Web API 层）
    ↓
frontend/（Next.js 前端展示层）
```

### 8.2 数据流

```
Tushare/聚宽 API  →  CSV 缓存  →  MarketFeed（逐日回放）
                                        ↓
                                  策略计算动量排名
                                        ↓
                         ┌──────────────┴──────────────┐
                    Headless 模式              Agent 模式
                   （直接执行）           （LLM 分析后执行）
                         ↓                       ↓
                    Portfolio 执行交易 ←─────────┘
                         ↓
                 PerformanceTracker 记录指标
                         ↓
                    TradingLogger 输出 JSON
                         ↓
                    FastAPI 提供 API
                         ↓
                    Next.js 仪表盘展示
```

### 8.3 关键设计模式

| 设计 | 说明 |
|------|------|
| **策略即纯函数** | `momentum.py` 中的函数无状态、无副作用，输入数据输出权重，方便测试和替换 |
| **工具桥接模式** | `TradingToolkit` 将引擎内部能力以 Tool 形式暴露给 LLM，是策略世界与 AI 世界的桥梁 |
| **双模式回测** | 同一个 Simulator 支持 Headless（快速验证）和 Agent（AI展示）两种模式 |
| **服务端启动即回测** | FastAPI 的 lifespan 钩子在服务启动时自动跑一次回测，Dashboard 无需等待 |

## 十一、总结

本项目实现了一个**从数据到展示的完整量化交易系统**，核心创新点在于将**传统动量轮动策略与 LLM Agent 结合**：

1. **策略层面**：采用经典的动量因子进行 ETF 轮动，逻辑简洁、有学术支撑
2. **AI 层面**：通过 Agno 框架让 LLM 调用工具自主决策，实现了"AI 交易员"的原型
3. **工程层面**：模块化架构、未来数据防泄漏、先卖后买执行、结构化日志、Web 可视化
4. **质量层面**：27 个单元测试，核心模块全覆盖

在半年回测期间取得了 **+11.03% 的总收益**，最大回撤仅 **-4.39%**，风险收益比良好。但需注意回测时段较短，策略的长期稳定性仍需更多数据验证。
